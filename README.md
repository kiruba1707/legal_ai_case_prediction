## 2. Dataset
Place your real file at `data/judgments.csv`. Assumed columns (rename via `utils/preprocessing.py` if different):
`case_id, court, judge_name, case_type, filing_date, decision_date, sections_involved, num_hearings, petitioner_type, respondent_type, case_description, outcome`
If missing, all scripts auto-generate synthetic data so the pipeline still runs.

## 3. Installation
```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

## 4. Execution Commands

### Day 1 ‚Äî ML Pipeline (Logistic Regression)
```bash
python data/generate_sample_data.py     # optional, only if no real judgments.csv
python day1_ml_pipeline/pipeline.py     # runs EDA -> FE -> Train -> Evaluate
```
Outputs: `models/logistic_regression.pkl`, `reports/evaluation_logistic_regression.json`, plots in `reports/`.

### Day 2 ‚Äî Advanced Models + Tuning + Streamlit
```bash
python day2_advanced_models/train_random_forest.py
python day2_advanced_models/train_xgboost.py
python day2_advanced_models/train_lightgbm.py
python day2_advanced_models/hyperparameter_tuning.py   # -> models/tuned_xgboost.pkl
streamlit run day2_advanced_models/app_streamlit_day2.py
```

### Day 3 ‚Äî Deep Learning + Deployment
```bash
python day3_deep_learning/train_mlp.py
python day3_deep_learning/train_cnn.py                 # -> models/cnn_model.pt
python day3_deep_learning/transfer_learning.py          # requires case_description text col

uvicorn day3_deep_learning.api_fastapi:app --reload --port 8000   # API: http://127.0.0.1:8000/docs
streamlit run day3_deep_learning/app_streamlit_day3.py             # calls the API
```
Run FastAPI and Streamlit in two separate terminals (from `legal_ai_pipeline/` root).

## 5. Day1 ‚Üí Day2 ‚Üí Day3 Integration Flow
1. **Day1** (`utils/preprocessing.py`) is the single source of truth for cleaning/feature engineering ‚Äî reused unchanged by Day2 and Day3, guaranteeing identical feature schema across all models.
2. `models/feature_columns.pkl` (saved in Day1/Day2) locks the feature order/names ‚Äî Day3's FastAPI and Streamlit app read this file to build the correct input vector for CNN/XGBoost inference.
3. **Day2** hyperparameter tuning produces `models/tuned_xgboost.pkl`, the model exposed later by Day3's `/predict/xgboost` endpoint ‚Äî so Day2's best model becomes a servable Day3 API route, not a separate artifact.
4. **Day3** CNN and MLP consume the same scaled tabular features (`StandardScaler` saved per-model) as Day1's Logistic Regression, so all four (LogReg, RF, XGB, LightGBM, MLP, CNN) are directly comparable in `reports/`.
5. **Day3 FastAPI** (`api_fastapi.py`) is the single inference layer; **Day3 Streamlit** (`app_streamlit_day3.py`) is a thin UI calling that API ‚Äî this is the deployment pattern the future Level 2 (RAG + Generative AI) layer will plug into as additional `/predict/*` or `/rag/*` routes on the same FastAPI app.
